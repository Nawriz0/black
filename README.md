# Blackjack — мультиплеер по комнатам

Онлайн-игра «21» против дилера: хост создаёт комнату с кодом, друзья подключаются по коду. Вся логика (колода, раздача, ход дилера, подсчёт очков) выполняется на сервере; клиент только отображает состояние и отправляет действия.

## Стек

- **Backend:** Node.js, Express, Socket.IO  
- **Frontend:** HTML, CSS, JavaScript (без React/TypeScript)

## Запуск локально

1. Установите [Node.js](https://nodejs.org/) (рекомендуется LTS, версия 18+).

2. В корне проекта выполните:

```bash
npm install
npm run start
```

3. Откройте в браузере: `http://localhost:3000`

По умолчанию сервер слушает порт **3000**. Его можно переопределить переменной окружения `PORT`.

## Переменные окружения

| Переменная | Назначение |
|------------|------------|
| `PORT` | Порт HTTP-сервера (по умолчанию `3000`) |

Других переменных для первой версии не требуется.

## Деплой backend (Render / Railway)

Идея: один процесс Node запускает и Express, и Socket.IO на **одном** origin — статика (`public/`) и WebSocket идут с одного хоста, отдельный «хостинг фронта» не нужен.

### Railway

1. Создайте проект из репозитория или загрузите папку.
2. Команда сборки: не обязательна (нет TS).
3. Команда старта: `npm run start` или `node server.js`.
4. Укажите `PORT`, если платформа не задаёт его сама (часто инжектится автоматически).
5. Откройте выданный URL — игра должна открываться с главной страницы.

### Render

1. **New** → **Web Service**, подключите репозиторий или загрузите файлы.
2. **Build command:** `npm install`  
3. **Start command:** `npm run start`
4. Render задаёт `PORT` — приложение уже читает `process.env.PORT` в `server.js`.
5. Включите поддержку WebSocket (у веб-сервисов Render она обычно работает для HTTP upgrade на том же порту).

### Важно для продакшена

- Друзьям отдавайте **HTTPS**-ссылку на ваш сервис — смешанный контент и блокировки WebSocket на части сетей реже возникают при одном домене.
- In-memory комнаты **пропадают при перезапуске** процесса (новый деплой, сон free-tier и т.д.) — это ожидаемо для MVP без БД.

## События Socket.IO (кратко)

Клиент → сервер: `create-room`, `join-room`, `rejoin-room`, `leave-room`, `start-game`, `hit`, `stand`, `restart-round`.

Сервер → клиент: `room-state-update`, `next-turn`, `dealer-turn`, `round-result`, `game-error`.
